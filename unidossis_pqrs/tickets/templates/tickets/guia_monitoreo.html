Identidad Visual del Monitoreo

Cada formulario ahora incluye un "Footer de Metadatos" premium que proporciona información crítica para la trazabilidad y auditoría en tiempo real.

F-01 · PORTAL-PQRS v1.0.0 Respuesta servidor: 45ms
Semáforo de Rendimiento: El sistema detecta automáticamente si el servidor responde lento. El color cambia a amarillo (>500ms) o rojo (>800ms) automáticamente.

Arquitectura del Sistema

La solución se basa en tres componentes que trabajan en cadena:

1. Middleware

Captura el tiempo exacto en que la petición llega al servidor Django.

request._inicio_form = time.time()

2. Context Processor

Identifica la URL actual, busca su código (F-01, etc.) y calcula la latencia del servidor.

3. Template Partial

Componente visual (_form_footer.html) que renderiza los datos e incluye telemetría del lado del cliente.

Guía de Uso y Mantenimiento

¿Cómo registrar un nuevo formulario?

Para añadir un nuevo formulario al sistema de auditoría, abra el archivo:

tickets/context_processors.py

Y añada una entrada al diccionario FORM_REGISTRY:

'nombre_de_la_ruta': { 'codigo': 'F-XX', 'nombre': 'NOMBRE-CORTO', 'version': 'v1.0.0' }

Inclusión en el HTML

En cualquier template nuevo, simplemente añada esta línea al final del formulario o contenedor principal:

{% include "tickets/_form_footer.html" %}
